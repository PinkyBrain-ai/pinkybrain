## ✨ PinkyBrainとは？

PinkyBrainは、マシンをピアツーピアのAIネットワークに接続します。マシン同士が通信し、計算資源とモデルを共有します。クラウド依存なし、中央サーバーなし、単一障害点なし。

**v5はパブリックメッシュを追加:** 共有CPU、RAM、GPU、AIモデルのグローバルネットワークに参加できます。プライベートネットワークはプライベートのまま。パブリックメッシュはオプトインの追加レイヤーです。

**簡単に言うと:** BitTorrentのようなもの、ただしAI向け。計算を共有すれば、50以上のモデルにアクセスできます。データは自分のマシンに残ります。エンドツーエンド暗号化。常に。

## 🚀 クイックスタート

### 前提条件
- Python 3.12+
- ローカルで動作する[Ollama](https://ollama.ai)（またはクラウドモデルエンドポイント）
- （オプション）自動ピア検出用[Tailscale](https://tailscale.com)

### インストールと実行

```bash
# クローン
git clone https://github.com/PinkyBrain-ai/pinkybrain.git
cd PinkyBrain

# 実行
python3 src/pinkybrain_v5.py

# または設定ファイルを指定
python3 src/pinkybrain_v5.py --config config/bug.json
```

### ネットワークの接続

```json
{
  "node_name": "my-node",
  "private": {
    "p2p_secret": "your-shared-secret-here",
    "peers": [
      {"name": "other-node", "host": "192.168.1.100", "port": 8080}
    ]
  },
  "public_mesh": {
    "enabled": false
  }
}
```

これだけです。プライベートネットワークはすぐに動作します。メッシュに参加しますか？`"enabled": true`を設定して共有内容を選択してください。

### 🤖 マルチLLMプロバイダー

- 型付きメッセージ：`query`、`memory_sync`、`memory_update`、`ping/pong`、`auth`
- 指数バックオフによる自動再接続

### 🔐 分散型認証
- **Ed25519アイデンティティ** — 各ノードが独自のキーペアを生成
- **HMAC共有シークレット** — プライベートネットワーク向けのよりシンプルな代替
- **信頼のウェブ** — ノードが相互に保証、推移的信用
- **ノードごとのレート制限**（トークンバケットアルゴリズム）
- **ステルスモード** — 非表示ノード、信頼できるピアのみ

### 🧠 分散メモリ（CRDT）
- **競合なし複製データ型** — マージ競合なし
- **ゴシッププロトコル** — 変更が自動伝播
- **ベクタークロック** — イベントの因果順序付け
- **TTLサポート** — エントリが自動的に期限切れ

### 🤖 AIモデルルーティング
- **ローカルモデル優先** — 可能な場合はローカルOllamaにクエリ
- **オンデマンドクラウドモデル** — `model:cloud`構文
- **ピアフェイルオーバー** — ローカルモデルがビジーな場合ピアにルーティング
- **アンサンブルコンセンサス** — 複数モデルにクエリ、最適な回答を返却
- **サーキットブレーカー** — ダウンしたピアへの過剰リクエストを防止

プライベートネットワーク（p2p_secret）とパブリックメッシュ（Ed25519 + 信頼のウェブ）は完全に分離されています。別ポート、別認証、データ漏洩ゼロ。

### 貢献ベースのクオータ

| 貢献 | スコア | パブリッククオータ |
|---|---|---|
| 共有なし | 0 | 1クエリ / 5分 |
| 1モデル共有 | +20 | 5クエリ / 分 |
| 2+モデル共有 | +30 | 20クエリ / 分 |
| 2GB RAM共有 | +20 | +10クエリ / 分 |
| GPU共有 | +20 | +20クエリ / 分 |
| 24時間稼働 | +10 | +5クエリ / 分 |

**より共有 = よりアクセス可能。** しかし、0共有でも5分ごとに1クエリ可能。誰もブロックされません。

## 🛡️ Resource Guard

あなたのマシンが最優先。Resource GuardはCPU/RAMを監視し、ビジー時にパブリック共有を自動一時停止します。

```python
class ResourceGuard:
    def can_accept_request(self) -> bool:
        cpu_usage = psutil.cpu_percent(interval=0.1)
        ram_usage = psutil.virtual_memory().percent
        
        if self.priority == "local_first":
            if cpu_usage > 70 or ram_usage > 85:
                return False  # ユーザーがビジー
        
        if cpu_usage > self.max_cpu + 40:
            return False
        
        return True
```

**ローカル優先は常に勝ちます。** マシンがビジーの場合、パブリックリクエストを拒否します。例外なし。

## 🧠 アダプティブスケジューラ

ネットワークは利用可能なピア数に基づいて最適な戦略を決定します。バージョン番号も手動モードも不要。

| 利用可能ピア | 戦略 | 能力 |
|---|---|---|
| 1–3 | シンプルルーティング | 1台のマシンで完全モデル |
| 4–10 | 部分シャーディング | モデルを2–4フラグメントに分割 |
| 11–50 | 完全シャーディング + 2×レプリケーション | パイプライン並列、冗長性 |
| 50+ | 分散RAID RAM | 仮想RAMディスク、3×レプリケーション、非同期プリフェッチ |

**移行は自動的に、中断なく行われます。** ピアが参加 → スケジューラが再配分。ピアが退出 → レプリカが引き継ぎ。何も気づきません。

## 💬 永続的会話ストア

あなたの会話はあなたのマシンに残ります。期間限定。

- **自動保存** — 各メッセージはローカルに保存。「保存」ボタン不要。
- **再開** — 明日PinkyBrainを開けば、会話がそのままあります。
- **検索** — キーワード、日付、モデル、タグで過去の会話を検索。
- **エクスポート** — Markdown、JSON、プレーンテキスト。あなたのデータ、あなたのフォーマット。
- **プライバシー** — プライベートP2Pで明示的に同期しない限り、会話はマシン外に出ません。
- **暗号化** — オプションのローカル暗号化。ディスクアクセスでも読めません。
- **トラッキングなし** — アナリティクスなし、データでのトレーニングなし。

### プライバシーレベル

| レベル | 何が起こるか | ユースケース |
|---|---|---|
| **プライベート**（デフォルト） | ローカルに留まり、同期なし | 個人的、機密 |
| **同期済み** | プライベートP2P経由でのみ同期 | デバイス間 |
| **共有** | 特定のピアと共有 | コラボレーション |
| **パブリック** | メッシュナレッジベースへのオプトイン貢献 | コミュニティ知識 |

**デフォルトはプライベート。常に。**

## 🔒 E2E暗号化

メッシュにクエリする際、データはエンドツーエンドで暗号化されます：

1. 質問はセッションキーで暗号化
2. パイプライン内の各ピアは自分のチャンクのみ復号、計算、再暗号化
3. あなただけが最終回答を復号可能

**各ピアが見られるもの：**
| データ | 可視？ | 理由 |
|---|---|---|
| 元の質問 | ❌ いいえ | セッションキーで暗号化 |
| 最終回答 | ❌ いいえ | セッションキーで暗号化 |
| 自分のチャンクの入出力テンソル | ✅ はい | 計算に必要 |
| 他のチャンクのデータ | ❌ いいえ | 他のピアのキーで暗号化 |

**これは約束ではありません。暗号学です。** すべてのメッシュピアが侵害されても、セッションキーなしではデータを読めません。セッションキーはあなたのマシンにのみ存在し、リクエスト中のみ。

## 📂 shared_models/ — プライベート/パブリックの境界

専用フォルダはあなたのモデルとパブリックメッシュ間の**唯一のインターフェース**です。

```
~/.pinkybrain/
├── conversations/        → 🔒 プライベート（共有なし）
├── memory/               → 🔒 プライベート（共有なし）
├── config/               → 🔒 プライベート（共有なし）
├── shared_models/        → 🌐 共有ゾーン（メッシュに可視）
│   ├── glm-5.1/          → ~/.ollama/models/glm-5.1へのシンボリックリンク
│   ├── llama3/           → コピーまたはシンボリックリンク
│   └── mistral/          → コピーまたはシンボリックリンク
└── ollama/               → 🔒 プライベートOllamaストレージ
```

```bash
pinkybrain share glm-5.1    # モデルを共有（シンボリックリンク作成）
pinkybrain unshare glm-5.1  # 共有停止（シンボリックリンクのみ削除）
pinkybrain shared            # 共有モデル一覧
```

**メッシュは`shared_models/`の外を絶対に読みません。** 共有停止は即座 — シンボリックリンク削除と同時にメッシュのアクセスが失われます。

## 🔒 セキュリティとプライバシー

- **プライベートネットワーク:** p2p_secretで暗号化（v4から変更なし）
- **パブリックメッシュ:** Ed25519アイデンティティ + TLSトランスポート
- **E2E暗号化:** 分散推論中のクエリはエンドツーエンド暗号化
- **プライベートとパブリックネットワーク間のデータ漏洩ゼロ**
- **パブリックリクエストはサンドボックス化:** プライベートメモリへのアクセスなし
- **Resource Guard:** PCがビジー時に共有を自動一時停止
- **ステルスモード:** 計算を共有しながらトラッカーには非表示
- **ゼロロギング:** メッシュピアはクエリやレスポンスを保存しません

## 🔄 マイグレーション（v4 → v5）
