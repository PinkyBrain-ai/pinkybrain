## ✨ PinkyBrain क्या है?

PinkyBrain मशीनों को पीयर-टू-पीयर AI नेटवर्क में जोड़ता है। आपकी मशीनें एक-दूसरे से बात करती हैं, कंप्यूट और मॉडल शेयर करती हैं — कोई क्लाउड निर्भरता नहीं, कोई केंद्रीय सर्वर नहीं, कोई एकल विफलता बिंदु नहीं।

**v5 सार्वजनिक मेश जोड़ता है:** साझा CPU, RAM, GPU और AI मॉडल के वैश्विक नेटवर्क में शामिल हों। आपका निजी नेटवर्क निजी रहता है। मेश एक अतिरिक्त परत है जिसे आप स्वैच्छिक रूप से शामिल होते हैं।

**संक्षेप में:** BitTorrent की तरह, लेकिन AI के लिए। आप कंप्यूट शेयर करते हैं, 50+ मॉडलों तक पहुँच पाते हैं। आपका डेटा आपकी मशीन पर रहता है। एंड-टू-एंड एन्क्रिप्टेड। हमेशा।

## 🚀 त्वरित शुरुआत

### आवश्यकताएं
- Python 3.12+
- स्थानीय रूप से चल रहा [Ollama](https://ollama.ai) (या क्लाउड मॉडल एंडपॉइंट)
- (वैकल्पिक) स्वचालित पीयर डिस्कवरी के लिए [Tailscale](https://tailscale.com)

### इंस्टॉल और चलाएं

```bash
# क्लोन करें
git clone https://github.com/PinkyBrain-ai/pinkybrain.git
cd PinkyBrain

# चलाएं
python3 src/pinkybrain_v5.py

# या कॉन्फिग फाइल के साथ
python3 src/pinkybrain_v5.py --config config/bug.json
```

### अपना नेटवर्क कनेक्ट करें

```json
{
  "node_name": "mera-node",
  "private": {
    "p2p_secret": "yaha-apna-shared-secret",
    "peers": [
      {"name": "dusra-node", "host": "192.168.1.100", "port": 8080}
    ]
  },
  "public_mesh": {
    "enabled": false
  }
}
```

बस। आपका निजी नेटवर्क तुरंत काम करता है। मेश जॉइन करना चाहते हैं? `"enabled": true` सेट करें और चुनें कि क्या शेयर करना है।

## 🔑 मुख्य सुविधाएं

### 🤖 मल्टी-LLM प्रोवाइडर
- **Ollama** (स्थानीय) — डिफ़ॉल्ट रूप से संगत
- **OpenAI** — GPT-4o, GPT-4o-mini, आदि
- **Anthropic** — Claude मॉडल
- **OpenAI-संगत** — LM Studio, vLLM, कोई भी कस्टम API
- सभी प्रोवाइडर के मॉडल P2P और मेश पर शेयर किए जाते हैं

### 🔌 WebSocket रियल-टाइम कम्युनिकेशन
- `/ws` एंडपॉइंट पर द्विदिश WebSocket
- टाइप किए गए संदेश: `query`, `memory_sync`, `memory_update`, `ping/pong`, `auth`
- घातीय बैकऑफ के साथ स्वतः रिकनेक्ट

### 🔐 विकेन्द्रीकृत प्रमाणीकरण
- **Ed25519 पहचान** — प्रत्येक नोड अपनी कीपेयर जनरेट करता है
- **HMAC शेयर्ड सीक्रेट** — निजी नेटवर्क के लिए सरल विकल्प
- **विश्वास जाल** — नोड्स एक-दूसरे की गारंटी लेते हैं, सकर्मक विश्वास
- **प्रति नोड दर सीमित** (टोकन बकेट एल्गोरिदम)
- **स्टेल्थ मोड** — छिपा हुआ नोड, केवल विश्वसनीय पीयर

### 🧠 वितरित मेमोरी (CRDT)
- **संघर्ष-मुक्त प्रतिकृति डेटा प्रकार** — कभी विलय संघर्ष नहीं
- **गॉसिप प्रोटोकॉल** — परिवर्तन स्वचालित रूप से प्रसारित होते हैं
- **वेक्टर घड़ियां** — घटनाओं का कारण संबंधी क्रम
- **TTL समर्थन** — प्रविष्टियां स्वचालित रूप से समाप्त होती हैं

### 🤖 AI मॉडल राउटिंग
- **स्थानीय मॉडल पहले** — संभव होने पर स्थानीय Ollama को क्वेरी
- **मांग पर क्लाउड मॉडल** — `model:cloud` सिंटैक्स
- **पीयर फेलओवर** — स्थानीय मॉडल व्यस्त होने पर पीयर को राउट करें
- **एनसेंबल कॉन्सेंसस** — कई मॉडल को क्वेरी करें, सबसे अच्छा उत्तर लौटाएं
- **सर्किट ब्रेकर** — मृत पीयर को बार-बार न मारें

### डुअल नेटवर्क आर्किटेक्चर

```
┌─────────────────────────────────────────────────────┐
│                    नोड (आप)                          │
│                                                     │
│  ┌─────────────┐          ┌──────────────────┐     │
│  │ निजी नेटवर्क │          │   सार्वजनिक मेश    │     │
│  │ p2p_secret  │          │   tracker         │     │
│  │ ┌─────────┐ │          │ ┌──────────────┐ │     │
│  │ │ Bug     │◄┼──P2P────┼─┤ नोड #42       │ │     │
│  │ └─────────┘ │          │ │ 2GB RAM      │ │     │
│  │ ┌─────────┐ │          │ │ 30% CPU      │ │     │
│  │ │ Pinky   │◄┼──P2P────┼─┤ Ollama स्थानीय│ │     │
│  │ └─────────┘ │          │ └──────────────┘ │     │
│  └─────────────┘          │ ┌──────────────┐ │     │
│                           │ │ नोड #789     │ │     │
│  ┌─────────────────┐      │ │ 8GB RAM      │ │     │
│  │ रिसोर्स गार्ड    │      │ │ RTX 4090     │ │     │
│  │ max_ram: 2GB    │      │ │ 4 मॉडल       │ │     │
│  │ max_cpu: 30%    │      │ └──────────────┘ │     │
│  │ gpu_share: बंद   │      │                  │     │
│  │ priority: स्थानीय │     │  Tracker:        │     │
│  └─────────────────┘      │  घोषणा/क्षमता   │     │
│                           └──────────────────┘     │

```

आपका **निजी नेटवर्क** (p2p_secret) सार्वजनिक मेश (Ed25519 + विश्वास जाल) से पूरी तरह अलग है। अलग पोर्ट, अलग प्रमाणीकरण, शून्य डेटा लीक।

### योगदान-आधारित कोटा

| योगदान | स्कोर | सार्वजनिक कोटा |
|---|---|---|
| कुछ शेयर नहीं | 0 | 1 क्वेरी / 5 मिनट |
| 1 मॉडल शेयर | +20 | 5 क्वेरी / मिनट |
| 2+ मॉडल शेयर | +30 | 20 क्वेरी / मिनट |
| 2GB RAM शेयर | +20 | +10 क्वेरी / मिनट |
| GPU शेयर | +20 | +20 क्वेरी / मिनट |
| 24घंटे अपटाइम | +10 | +5 क्वेरी / मिनट |

**अधिक शेयर = अधिक एक्सेस।** लेकिन शून्य शेयरिंग के साथ भी, आप हर 5 मिनट में 1 क्वेरी पा सकते हैं। किसी को ब्लॉक नहीं किया जाता।

## 🛡️ रिसोर्स गार्ड

आपकी मशीन पहले आती है। रिसोर्स गार्ड CPU/RAM की निगरानी करता है और आप व्यस्त होने पर सार्वजनिक शेयरिंग को स्वचालित रूप से रोकता है।

```python
class ResourceGuard:
    def can_accept_request(self) -> bool:
        cpu_usage = psutil.cpu_percent(interval=0.1)
        ram_usage = psutil.virtual_memory().percent
        
        if self.priority == "local_first":
            if cpu_usage > 70 or ram_usage > 85:
                return False  # उपयोगकर्ता व्यस्त है
        
        if cpu_usage > self.max_cpu + 40:
            return False
        
        return True
```

**स्थानीय प्राथमिकता हमेशा जीतती है।** यदि आपकी मशीन व्यस्त है, तो यह सार्वजनिक अनुरोधों को अस्वीकार करती है। बिना किसी अपवाद के।

## 🧠 एडैप्टिव शेड्यूलर

नेटवर्क उपलब्ध पीयर की संख्या के आधार पर सबसे अच्छी रणनीति चुनता है। कोई संस्करण संख्या नहीं, कोई मैनुअल मोड नहीं।

| उपलब्ध पीयर | रणनीति | क्षमता |
|---|---|---|
| 1–3 | सरल राउटिंग | एक मशीन पर पूर्ण मॉडल |
| 4–10 | आंशिक शार्डिंग | मॉडल 2–4 टुकड़ों में विभाजित |
| 11–50 | पूर्ण शार्डिंग + 2× प्रतिकृति | पाइपलाइन समानांतर, अतिरेक |
| 50+ | वितरित RAID RAM | वर्चुअल RAM डिस्क, 3× प्रतिकृति, एसिंक्रोनस प्रीफेच |

**संक्रमण स्वचालित और बिना व्यवधान होते हैं।** एक पीयर जुड़ता है → शेड्यूलर पुनः वितरित करता है। एक पीयर छोड़ता है → प्रतिकृतियां काम संभालती हैं। आप कुछ नहीं नोटिस करते।

## 💬 स्थायी कन्वर्सेशन स्टोर

आपकी बातचीत आपकी मशीन पर रहती है। बिंदु।

- **स्वतः सेव** — हर संदेश स्थानीय रूप से सेव। कोई "सेव" बटन नहीं।
- **रिज्यूम** — कल PinkyBrain खोलें, आपकी बातचीत वहाँ है।
- **खोज** — कीवर्ड, तारीख, मॉडल या टैग से कोई भी पिछली बातचीत खोजें।
- **एक्सपोर्ट** — Markdown, JSON, सादा टेक्स्ट। आपका डेटा, आपका फॉर्मेट।
- **प्राइवेसी** — बातचीत कभी आपकी मशीन से बाहर नहीं जाती जब तक आप P2P के माध्यम से सिंक न करें।
- **एन्क्रिप्शन** — वैकल्पिक स्थानीय एन्क्रिप्शन। डिस्क एक्सेस भी उन्हें नहीं पढ़ सकता।
- **कोई ट्रैकिंग नहीं** — कोई एनालिटिक्स नहीं, आपके डेटा पर कोई ट्रेनिंग नहीं।

### प्राइवेसी स्तर

| स्तर | क्या होता है | उपयोग केस |
|---|---|---|
| **निजी** (डिफॉल्ट) | स्थानीय रहता है, कभी सिंक नहीं | व्यक्तिगत, संवेदनशील |
| **सिंक किया** | केवल निजी P2P के माध्यम से सिंक | आपके डिवाइस के बीच |
| **शेयर किया** | विशिष्ट पीयर के साथ शेयर | सहयोग |
| **सार्वजनिक** | मेश नॉलेज बेस में ऑप्ट-इन योगदान | सामुदायिक ज्ञान |

**डिफॉल्ट निजी है। हमेशा।**

## 🔒 E2E एन्क्रिप्शन

जब आप मेश को क्वेरी करते हैं, तो आपका डेटा एंड-टू-एंड एन्क्रिप्टेड होता है:

1. आपका प्रश्न सेशन कुंजी से एन्क्रिप्ट होता है
2. पाइपलाइन में प्रत्येक पीयर केवल अपना टुकड़ा डिक्रिप्ट करता है, गणना करता है, फिर से एन्क्रिप्ट करता है
3. केवल आप अंतिम उत्तर डिक्रिप्ट कर सकते हैं

**प्रत्येक पीयर क्या देख सकता है:**
| डेटा | दृश्यमान? | क्यों |
|---|---|---|
| आपका मूल प्रश्न | ❌ नहीं | आपकी सेशन कुंजी से एन्क्रिप्टेड |
| अंतिम उत्तर | ❌ नहीं | आपकी सेशन कुंजी से एन्क्रिप्टेड |
| अपने टुकड़े के इनपुट/आउटपुट टेंसर | ✅ हाँ | गणना के लिए आवश्यक |
| अन्य टुकड़ों का डेटा | ❌ नहीं | अन्य पीयर की कुंजियों से एन्क्रिप्टेड |

**यह एक वादा नहीं है। यह क्रिप्टोग्राफी है।** भले ही सभी मेश पीयर समझौता हो गए हों, वे आपकी सेशन कुंजी के बिना आपका डेटा नहीं पढ़ सकते — जो केवल आपकी मशीन पर मौजूद है, केवल अनुरोध की अवधि के लिए।

## 📂 shared_models/ — निजी/सार्वजनिक सीमा

एक समर्पित फोल्डर जो आपके मॉडल और सार्वजनिक मेश के बीच **एकमात्र इंटरफेस** है।

```
~/.pinkybrain/
├── conversations/        → 🔒 निजी (कभी शेयर नहीं)
├── memory/               → 🔒 निजी (कभी शेयर नहीं)
├── config/               → 🔒 निजी (कभी शेयर नहीं)
├── shared_models/        → 🌐 शेयरिंग ज़ोन (मेश को दृश्यमान)
│   ├── glm-5.1/          → symlink से ~/.ollama/models/glm-5.1
│   ├── llama3/           → कॉपी या symlink
│   └── mistral/          → कॉपी या symlink
└── ollama/               → 🔒 निजी Ollama स्टोरेज
```

```bash
pinkybrain share glm-5.1    # मॉडल शेयर करें (symlink बनाता है)
pinkybrain unshare glm-5.1  # शेयरिंग बंद करें (केवल symlink हटाता है)
pinkybrain shared            # शेयर किए गए मॉडल सूचीबद्ध करें
```

**मेश कभी `shared_models/` के बाहर नहीं पढ़ता।** शेयरिंग बंद करना तत्काल है — symlink हटाते ही मेश की पहुँच समाप्त हो जाती है।

## 🔒 सुरक्षा और गोपनीयता

- **निजी नेटवर्क:** p2p_secret के साथ एन्क्रिप्टेड (v4 से अपरिवर्तित)
- **सार्वजनिक मेश:** Ed25519 पहचान + परिवहन के लिए TLS
- **E2E एन्क्रिप्शन:** वितरित अनुमान के दौरान क्वेरी एंड-टू-एंड एन्क्रिप्टेड
- **निजी और सार्वजनिक नेटवर्क के बीच शून्य डेटा लीक**
- **सार्वजनिक अनुरोध सैंडबॉक्स:** निजी मेमोरी तक कोई पहुँच नहीं
- **रिसोर्स गार्ड:** आपका PC व्यस्त होने पर शेयरिंग स्वतः रोकें
- **स्टेल्थ मोड:** कंप्यूट शेयर करें लेकिन tracker पर छिपे रहें
- **शून्य लॉगिंग:** मेश पीयर कभी क्वेरी या प्रतिक्रियाएं संग्रहित नहीं करते

## 🔄 माइग्रेशन (v4 → v5)

1. v5, v4 के साथ **पिछड़े संगत** है
2. निजी नेटवर्क कॉन्फिग ठीक पहले की तरह काम करती है
3. `public_mesh` सेक्शन **वैकल्पिक** है — डिफॉल्ट रूप से अक्षम
4. मौजूदा v4 नोड्स निजी नेटवर्क पर v5 नोड्स के साथ बात कर सकते हैं
5. सार्वजनिक मेश **ऑप्ट-इन** है: `public_mesh.enabled = true` सेट करें
